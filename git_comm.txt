Коммит - сохранение конкретной версии файла внутри Git

$ git status	---------	Просмотреть какие файлы закоммичены


$ git add <NameOfFile> / $ git add .	---------	Подготавливает данные файлы к следующему коммиту, добавляя их в "индекс" гита.
													"Индекс" гита - хранилище файлов, готовых к коммиту


$ git commit -m "{message}"		----	Запоминает версию(коммит) всех файлов в репозитории из индекса и присваивает ей уникальный код(хэш), по которому затем к этой версии можно вернуться 


$ git log	---		Список всех коммитов


$ git cheсkout <хэш коммита>	---		Откатить к коммиту с хэшем: <хэш коммита>


$ git push <имя репозитория> <ветка>	---		Запушить(закинуть) файл в удаленную директорию 

!!!Algo:!!! КАК ПУШИТЬ ФАЙЛЫ НА GitHub !!!!!!
 1. git status -- смотрим файлы 
 2. git add <NameOfFile> -- подготавливает файлы к следующему коммиту
 3. git commit -m "{message}" -- делаем коммит
 4. git log -- берем хэш коммита из строки commit <хэша> (HEAD -> master)
 5. git branch -- проверить, на какой ветке мы сейчас, если не в той, где нужно то:
	5.1 git cheсkout <имя ветки> -- переходим в ветку с именем <имя ветки>
	5.2 Опять git branch -- если в нужной ветке, то goto 6, иначе goto5.1.
 6. git push <имя репозитория> <ветка> -- пушим файл в репозиторий на ветке.
 7. Проверяем GitHub.
